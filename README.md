# منصة نشر الأبحاث للمحامين ودكاترة الجامعة

## نظرة عامة
هذا المشروع يحتوي على:
- واجهة مستخدم HTML/CSS/JS
- خادم Node.js + Express
- تخزين بيانات الأبحاث بـ SQLite
- رفع ملفات بحجم حتى 10 ميجابايت
- عرض الأبحاث وحذفها عبر API

## تشغيل محليًا
1. افتح مجلد المشروع:
   ```powershell
   cd "c:\Users\Dr Ahmed\Downloads\vs\index.html\lawyers-doctors-site"
   ```
2. ثبت الحزم:
   ```powershell
   npm install
   ```
3. شغل الخادم:
   ```powershell
   npm start
   ```
4. افتح المتصفح على:
   ```text
   http://localhost:3000
   ```

## استضافة حقيقية مقترحة
### الخيار الأفضل الآن: Render.com
1. أنشئ حساب مجاني في [Render](https://render.com)
2. ارفع المشروع إلى GitHub أو GitLab
3. أنشئ خدمة ويب جديدة `Web Service`
4. ربط المشروع بالمستودع
5. إعدادات الخدمة:
   - Build Command: `npm install`
   - Start Command: `npm start`
6. Render يعطيك رابطًا حقيقيًا للموقع

## تشغيل محلي مع ngrok
إذا تريد أن يصل أي جهاز إلى الخادم الموجود على جهازك:
1. شغّل الخادم:
   ```powershell
   npm start
   ```
2. في نافذة PowerShell أخرى، شغّل:
   ```powershell
   npx ngrok http 3000
   ```
3. أو استخدم الملف الجاهز:
   ```powershell
   .\start-server-ngrok.ps1
   ```
4. انسخ الرابط الذي يعطيه `ngrok` وأرسله لأي شخص.

### ملاحظة مهمة
الخادم الحالي يخزن الملفات في مجلد `uploads/` وبيانات SQLite في `research.db` داخل نفس الخدمة. هذا يعمل في البداية، لكن لسهولة واستمرارية أفضل في الإنتاج يفضل:
- استخدام قاعدة بيانات سحابية مثل PostgreSQL أو MongoDB Atlas
- تخزين الملفات في خدمة تخزين سحابية مثل S3 أو Firebase Storage

## ملاحظات تقنية
- الخادم يستمع على `PORT` المقدم من البيئة، لذا يعمل على أي خدمة استضافة تدعم Node.js
- ملفات المشروع التي يجب رفعها كلها موجودة حاليًا في هذا المجلد
